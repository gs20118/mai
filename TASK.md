## 2. 대회/과제 개요
 
### 2.1 핵심 임무
 
본선 과제는 드론 정찰, 영상 데이터 수집, 인공지능 기반 객체 인식 및 분석 기술을 결합하여 모의 군사 임무 수행 능력을 평가하는 과제이다.
 
적의 공습으로 피해가 발생한 공군 비행장을 가상 전장 환경, 즉 디오라마 경기장으로 구현한다. 참가팀은 드론 기반 정찰(ISR: Intelligence, Surveillance, Reconnaissance) 자산과 인공지능(AI) 기술을 활용하여 피해 상황을 신속하고 정확하게 식별하고, 지휘통제체계(C2)에 보고하는 임무를 수행한다.
 
### 2.2 임무의 큰 축
 
1. 모의 작전 환경에서의 정찰 비행: 드론 공중 정찰 임무
2. 드론 영상 기반 인공지능 객체 인식: AI 기반 정찰 객체 탐지
3. 상황 분석 및 임무 보고: 분석 결과 기반 상황 판단
### 2.3 임무 평가 기준 요약
 
제한된 체공 시간 내에 드론으로 수집한 영상 데이터를 기반으로 객체 탐지, 영상 분류, 위치 인식 및 생성형 AI 기술을 통합하여 결과물을 도출하고, 이를 공군의 AI 기반 전투피해평가(BDA) 및 정찰(ISR) 임무 수행 방식으로 검증한다.

## 3. 임무 내용
 
### 3.1 전체 임무 수행 내용
 
본선 임무는 크게 다음 항목으로 구성된다.
 
1. 활주로 폭파구 탐지
   - 활주로와 유도로의 폭파구 크기 및 위치 탐지
   - 활주로 폭파구 개수 탐지 및 가용 길이 산출
   - 폭파구 데이터셋 구축 및 라벨링
   - 객체 탐지 모델 개발: 객체 위치 산출, 크기/종류 분류
   - 활주로 가용 길이 예측
2. 시설물 피해 진단/확인
   - 시설물의 정상/파손/화재 상태 분류
   - 시설물 상태 데이터셋 구축 및 분류 모델 개발
   - 시설물 피해 판정 알고리즘 구현
3. 불발탄/위험 장애물 탐지
   - 불발탄 탐지 대상: 미사일, 포탄, 자탄
   - 데이터셋 구축 및 라벨링
   - 객체 위치 및 개수 산출
4. 작전 상황 종합 분석
   - LLM 기반 AI 에이전트 활용
   - 자동 상황 보고서 생성
5. 임무 완료 소요 시간
   - 드론 정찰, 데이터 전송, AI 추론, JSON 임무 코드 전송까지의 전체 흐름을 제한 시간 내에 수행

## 4. 세부 임무 내용
 
### 4.1 폭파구 탐지
 
- 설명: 드론 영상 또는 이미지 데이터를 활용하여 활주로 및 유도로에 배치된 폭파구를 탐지한다.
- 수행 내용:
  - 폭파구 위치 식별: 구간 기준으로 식별
  - 폭파구 크기 분류: 대형, 중형, 소형
  - 폭파구 개수 탐지
  - 활주로 가용 길이 산출
- 평가 요소:
  - 폭파구 위치 식별 정확도
  - 폭파구 크기 분류 정확도
  - 폭파구 개수 정확도
  - 활주로 가용 길이 산출 정확도
- 주요 기술:
  - 객체 탐지
  - 이미지 세그멘테이션
  - ArUco 기반 스케일 보정
  - 면적 및 직경 계산
  - 위치 좌표 추정
### 4.2 시설물 피해 확인
 
- 설명: 공항 주요 시설물의 상태를 분석하여 분류한다.
- 대상: 관제탑, 관제레이더, 격납고, 일반건물/행정동 계열 시설, 일반건물/보급창고 계열 시설, 무기고 등 총 6개 시설물
- 상태 분류:
  - 정상
  - 파손
  - 화재
- 평가 요소:
  - 시설물 식별 정확도
  - 상태 분류 정확도
  - 화재 탐지 정확도
  - 시설물 누락 여부
- 주요 기술:
  - 이미지 분류
  - 객체 탐지
  - 파손/화재 인식
  - AI 기반 상태 분류
### 4.3 불발탄/위험 장애물 탐지
 
- 설명: 공항 내 불발탄을 탐지하고 종류를 분류한다.
- 탐지 대상:
  - 미사일
  - 포탄
  - 자탄
- 수행 내용:
  - 불발탄 위치 탐지
  - 불발탄 종류 분류
  - 불발탄 개수 산출
- 평가 요소:
  - 불발탄 탐지 정확도
  - 종류 분류 정확도
  - 위치 식별 정확도
  - 개수 산출 정확도
- 주요 기술:
  - 객체 탐지
  - 다중 객체 분류
  - 위치 추정
  - 객체 카운팅
### 4.4 LLM 기반 상황 보고
 
- 설명: 임무 결과를 기반으로 LLM을 활용하여 상황 보고서를 생성한다.
- 보고서에 반영되어야 하는 항목:
  - 보고 시간
  - 활주로 구역 내 폭파구 총 개수
  - 활주로 가용 길이
  - 활주로 상태 및 운용 여부
  - 화재 시설물 등 확인된 피해 정보
- 주요 기술:
  - LLM 연동
  - JSON 데이터 처리
  - 상황 정보 구조화
  - 자동 보고서 생성
### 4.5 임무 수행 시간
 
- 설명: 드론 이륙부터 최종 착륙 완료까지의 총 소요 시간을 측정한다.
- 제한 시간: 3분, 즉 180초
- 평가 요소:
  - 임무 완료 소요 시간
  - 제한 시간 내 임무 수행 여부
  - 데이터 전송 성공 여부
- 주요 기술:
  - 영상 수집 효율화
  - 시간 데이터 전송
  - AI 기반 영상 분석
  - 대시보드 자동 보고

## 5. 전체 임무 수행 프로세스
 
### 5.1 경기 시간 구조
 
팀당 경기 시간은 총 5분이다.
 
- 대기 시간: 1분
- 준비 시간: 1분
  - 코드 수정 및 프로그램 시작 가능
- 임무 수행 시간: 3분, 최대 180초
### 5.2 전체 흐름
 
1. 사전 점검 및 대기 장소 이동
2. 출발 준비
3. 경기 시작
4. 드론 원격 제어
5. 임무 수행
6. 데이터 전송
7. 정찰 결과 보고
### 5.3 임무 코드 수행 프로세스
 
1. 이륙 및 임무 시작
   - 준비 단계 코드 전송: `start.json`
2. 활주로/유도로 폭파구 탐지
   - 폭파구 위치 식별
   - 폭파구 크기 분류
   - 관련 전송 파일: `crater_detect.json`
3. 활주로 폭파구 개수 산출
   - 활주로 구간의 폭파구 총 개수 산출
   - 관련 전송 파일: `crater_count.json`
4. 활주로 가용 길이 산출
   - 폭파구 위치를 바탕으로 가용 활주로 길이 산출
   - 관련 전송 파일: `runway_status.json`
5. 시설물 상태 확인
   - 주요 시설물 상태 식별
   - 정상/파손/화재 분류
   - 관련 전송 파일: `facility_status.json`
6. 불발탄 탐지
   - 활주로/유도로 내 불발탄 탐지
   - 불발탄 위치 및 종류 식별
   - 관련 전송 파일: `uxo_detect.json`
7. 불발탄 개수 확인
   - 활주로 구간 내 불발탄 총 개수 산출
   - 관련 전송 파일: `uxo_count.json`
8. LLM 기반 상황 보고
   - 모든 분석 결과 통합
   - LLM 기반 작전 상황 보고서 자동 생성
   - 관련 전송 파일: `report.json`
9. 분석 결과 전송 완료 확인
10. 임무 종료 및 착륙
11. 임무 시간 측정 종료

## 7. 대회 환경
 
### 7.1 전체 시스템 흐름
 
1. 드론 정찰
2. 촬영 영상 또는 이미지 저장
3. 주최 측 제공 전송 프로그램을 통해 노트북으로 자동 전송
4. 참가팀이 개발한 AI 모델로 분석
5. 임무별 JSON 코드 생성
6. 대회 대시보드/관제 서버로 전송
### 7.2 제공 드론
 
- 모델: DJI NEO 2
- 구성: DJI NEO 2 본체, 배터리 2개, 액세서리, 전용 가방
- 비행 시간: 약 20분
- 무게: 약 151g
- 통신: Wi-Fi
- 카메라: 4K 60fps video
- 최대 속도: 약 12m/s
- 짐벌: 2-axis
- 저장공간: 약 49GB
- 특징: 4K 고화질 이미지/영상 촬영 가능
### 7.3 경기장 구성
 
- 전체 크기: 5,000mm(가로) × 4,000mm(세로) × 500mm(높이)
- 주요 구성: 경기장 트랙, 테이블, 시설물, 폭파구, 미션용 객체, 미션장치 등
- 축척: 1:600
- 구역 구분:
  - 시설물 구역
  - 유도로 구역 A/B
  - 활주로 구역
- 건물 구성: 관제탑, 격납고, 무기고, 레이더시설, 보급창고, 행정동/일반건물 계열 시설
- 도로 구성: 활주로, 유도로, 일반도로
- 테스트 경기장: 2세트 제공
  - 테이블, 미션장치, 환경 디오라마를 제외하고 메인 경기장과 동일한 구성
### 7.4 경기장 구역/구간
 
| 구역 | 구간 | 크기(W×D) | 개수 |
|---|---|---:|---:|
| 시설물 구역 | FA-01 ~ FA-06 | 160cm × 80cm | 4개 |
| 시설물 구역 | FA-02, FA-05 | 180cm × 80cm | 2개 |
| 유도로 A | TW-A1 ~ TW-A5 | 100cm × 80cm | 5개 |
| 활주로 | RW-01 ~ RW-10 | 50cm × 80cm | 10개 |
| 유도로 B | TW-B1 ~ TW-B5 | 100cm × 80cm | 5개 |
 
추가 설명:
 
- 구역은 경기장의 대분류 영역을 의미한다.
- 구간은 각 구역을 세분화한 위치 식별 단위이다.
- 예시: 구역은 활주로, 유도로, 시설물 구역이고, 구간은 RW-01, FA-02 등이다.
- 경기장 규모: 500cm × 400cm
- 경기장 축척: 1:600
- 실제 환경 규모: 약 3,000m × 2,400m
- 예시: 활주로 RW-01 구역의 실제 길이는 300m
- 가용 길이 산출을 위해 실제 환경 규모를 산출해야 한다.

## 8. 미션 객체
 
### 8.1 폭파구
 
폭파구는 대형, 중형, 소형 3종으로 구성된다.
 
| 구분 | 내용 |
|---|---|
| 종류 | 대형, 중형, 소형 |
| 대형 크기 | 179 × 200 × 30 mm |
| 중형 크기 | 159 × 150 × 23 mm |
| 소형 크기 | 102.5 × 99 × 16 mm |
| 재질 | 3D 프린팅(PLA) |
| 색상 | 검정 |
 
### 8.2 시설물
 
시설물 디오라마는 총 6종으로 구성된다.
 
- 관제탑
- 관제레이더
- 격납고
- 일반건물 1
- 일반건물 2
- 무기고
문서 내 표기상 행정동, 보급창고 등의 명칭도 시설물 구성 설명에 포함되어 있으나, 이미지 예시는 위 6종으로 표시되어 있다.
 
시설물 관련 정보:
 
| 항목 | 내용 |
|---|---|
| 종류 | 관제탑, 관제레이더, 격납고, 일반건물/행정동 계열, 일반건물/보급창고 계열, 무기고 등 총 6종 |
| 상태 유형 | 정상, 파손, 화재 상태 변화 |
| 크기 | 바닥 크기 약 30~45cm, 높이 약 30~50cm |

## 10. 공통 운영 규칙
 
### 10.1 경기 운영
 
#### 경기 순서
 
- 참가팀의 경기 순서는 오리엔테이션 시 별도 프로그램을 통해 결정한다.
- 결정된 경기 순서는 1차 평가와 2차 평가에 동일하게 적용한다.
- 참가팀은 지정된 순서에 따라 경기에 참여해야 한다.
#### 경기 횟수
 
- 본선 평가는 팀별 총 2회 실시한다.
- 최종 점수는 2회 평가 결과 중 고득점 1회를 기준으로 정한다.
#### 경기 시간
 
- 팀당 경기 시간은 대기 시간, 준비 시간, 임무 시간으로 나뉘며 총 5분(300초)이다.
- 경기 전 대회 설명 및 이동 대기 시간 1분, 준비 시간 1분을 제공한다.
- 임무 시간은 3분(180초)이며 드론 이륙 시점부터 최종 착륙 완료 시점까지를 기준으로 측정한다.
- 드론 착륙장에 설치된 자동 센서 및 타이머 시스템을 공식 시간 기준으로 적용한다.
- 임무 시간 3분을 초과하면 즉시 경기가 종료되며, 중단 시점까지 대시보드에 전송된 데이터만 평가한다.
- 드론 추락 또는 충돌 등으로 경기장 시설물이 파손되거나 경기 진행이 불가능하다고 판단될 경우 심사위원은 경기를 즉시 중단할 수 있다.
### 10.2 경기 진행
 
#### 대기 시간: 1분
 
- 참가팀은 지정된 위치로 이동하여 대기한다.
- 팀원 이동, 노트북 설치, 드론 이륙장 배치를 수행한다.
- 대기 시간 동안 프로그램 실행 및 조작은 금지한다.
#### 준비 시간: 1분
 
- 현장 심사위원의 경기 시작 신호 후 준비 시간을 부여한다.
- 드론 조종 앱 실행, AI 프로그램 실행, 장비 점검이 가능하다.
- 준비 시간 내 드론 및 시스템 점검을 완료해야 한다.
- 준비 시간 종료 전 드론 이륙을 완료해야 하며, 이륙과 동시에 경기 시간이 시작된다.
- 1분 초과 후 30초 이내에도 이륙이 지연되는 경우 자동 실격 처리된다.
#### 임무 시간: 3분
 
- 임무 시간은 드론 이륙 시점부터 최종 착륙 완료 시점까지를 기준으로 측정한다.
- 드론 이륙 후에는 프로그램 조작, 설정 변경, 재실행을 할 수 없다.
- 드론 이륙 후 노트북 및 전자장치 사용은 금지된다.
- 단, 드론 비행을 위해 제공된 스마트기기는 예외이다.
- 제공된 앱 외 다른 앱 사용은 불가하다.
- 경기 중 프로그램 또는 전자장치를 만지거나 조작하면 1회 경고한다.
- 동일 행위가 2회 적발되면 경기는 즉시 중단되며, 중단 시점까지 수집된 데이터만 평가에 반영한다.
- 고의적인 규정 위반으로 판단될 경우 심사위원회 판단에 따라 실격 처리될 수 있다.
### 10.3 장비 규칙
 
#### 제공 장비 사용
 
- 모든 참가팀은 주최 측이 제공한 드론 및 부속 장비를 사용해야 한다.
- 제공된 장비의 성능을 임의로 교체, 분해, 개조 또는 추가 장착할 수 없다.
- 모든 임무 수행 및 데이터 처리는 참가팀에게 제공된 노트북에서 수행해야 한다.
#### 추가 장비 사용
 
- 데이터 수집 또는 성능 향상을 위한 별도 장치, 센서, 기구물 등을 사용할 경우 사전에 주최 측의 승인을 받아야 한다.
- 승인되지 않은 장비 사용은 엄격히 금지한다.
#### 경기장 환경
 
- 본선 경기장과 연습 경기장은 동일한 규격의 트랙 및 미션 객체로 구성한다.
- 단, 경기 진행에 직접 관련되지 않는 일부 구성품, 예를 들어 드론 액세서리, 배터리, 안전시설 등은 상이할 수 있다.
- 연습 경기장은 팀별 사용 시간에 따라 시설물 및 객체의 위치와 상태가 바뀔 수 있으나, 경기장 외부로 이동할 수 없다.
- 연습 종료 후 시설물 및 객체를 원위치로 복구해야 한다.
- 공동 사용 시설이므로 청결 유지 및 파손 방지에 유의해야 한다.
- 본선 경기장은 모든 팀에게 동일한 사전 활용 시간을 제공한다.
- 팀별 사용 시간 외 본선 경기장 출입 및 사진/동영상 촬영 등 데이터 수집 행위는 제한된다.
- 본선 경기장의 시설물 및 객체 배치는 임의로 변경할 수 없다.
- 대회 진행 시 조명 및 경기장 주변 환경은 외부 요인에 의해 변경될 수 있다.
### 10.4 프로그램 사용 규정
 
#### 소프트웨어 개발
 
- 참가팀은 제한 사항 외 모든 소프트웨어 환경에서 자유롭게 프로그램을 개발할 수 있다.
- 참가팀은 AI 모델, 영상처리 알고리즘, 데이터 분석 기능 등을 자유롭게 구현할 수 있다.
#### 제한 사항
 
사용이 제한되는 항목:
 
- 외부 GPU 서버
- 클라우드 기반 AI 서비스
- 원격 컴퓨팅 자원
- 가상 머신(VM)
- 타 참가팀 또는 외부 시스템의 실시간 지원 등
#### 데이터 및 협업 규정
 
- 참가팀은 다른 참가팀과 데이터, 프로그램, 학습모델, 소스코드 및 장비를 공유하거나 교환할 수 없다.
- 공동 개발 또는 기술 지원 행위는 금지한다.
- 대회 일정 동안 외부 인원으로부터 기술적 지원을 받을 수 없다.
#### 프로그램 수정 규정
 
- 대회 평가가 시작된 이후에는 모든 참가팀의 프로그램 수정, AI 모델 변경, 설정 변경 및 재컴파일을 금지한다.
- 단, 경기 시작 전 각 팀에게 제공되는 1분의 준비 시간에는 프로그램 수정 및 점검이 가능하다.
- 경기 진행 중 프로그램 수정 사실이 확인될 경우 실격 처리할 수 있다.
- 1차 평가 종료 후 2차 평가 시작 전 모든 참가팀에게 약 10분 이내의 프로그램 수정 및 보완 시간을 제공한다.
- 참가팀은 심사위원 또는 운영진 요청 시 수정 내용을 설명해야 한다.
- 모든 참가팀은 대회 종료 후 1차, 2차 평가에 사용된 최종 결과물을 제출해야 한다.
- 제출된 결과물은 규정 준수 여부 및 평가 결과 검증 목적으로 활용된다.
### 10.5 감점 및 실격
 
다음 행위는 감점 또는 실격 대상이다.
 
- 승인되지 않은 장비 사용
- 규정 외 데이터 사용
- 대회 진행 중 승인 없는 소프트웨어 수정
- 경기 진행 방해 행위
- 경기 중 고의적 전자장치 사용
- 안전 규정 위반
- 하드웨어 임의 개조 또는 교체
- 타 참가팀과 데이터, 프로그램, 모델 공유
- 경기 결과 조작 시도
- 심사위원 또는 운영진의 정당한 지시 불이행
- 경기 중 고의적인 시설 파손 행위
- 기타 심사위원회가 중대한 규정 위반으로 판단한 행위
제재 원칙:

- 위반 행위의 고의성, 정도, 경기 결과에 미치는 영향을 종합적으로 고려하여 제재 수위를 결정한다.
- 감점 및 실격 여부는 현장 심사위원회의 최종 결정에 따른다.

## 11. 종목별 규칙
 
### 11.1 공통 드론 운용 규칙 1
 
#### 일반 사항
 
- 드론 조종은 주최 측에서 제공한 스마트폰 및 사전 설치된 드론 제어 앱만 사용할 수 있다.
- 드론에 장착된 안전가드는 분리하거나 개조할 수 없다.
- 배터리 교체, 기체 점검 및 유지관리는 준비 시간 내에 완료해야 한다.
- 드론은 데이터 수집 목적으로만 활용해야 한다.
- 사전에 정의된 좌표 또는 순차 동작만으로 결과를 생성하는 방식은 제한된다. 이는 감점 및 실격 사유가 될 수 있다.
#### 비행 제한
 
- 대회 준비 기간 동안 지정된 곳에서만 드론을 운용해야 한다.
- 대회 진행 시 경기장 외부로 이탈한 경우, 현장 심사위원의 지시에 따라 즉시 복귀 또는 이동 비행해야 한다.
- 드론이 제공하는 인텔리전트 기능은 사용할 수 없다.
  - 예: 셀프 캡처, 슬로모션, Dolly Zoom, QuickShot, MasterShot, 제스처 컨트롤 등
- 드론 짐벌 기능은 사용할 수 있다.
### 11.2 공통 드론 운용 규칙 2
 
#### 드론 임무 수행
 
- 촬영한 사진 또는 영상은 드론 앱을 통해 저장한다.
- 저장된 데이터는 주최 측이 제공하는 전송 프로그램을 통해 노트북으로 자동 전송된다.
- 전송된 데이터는 참가팀이 개발한 AI 프로그램을 통해 자동 분석되어야 한다.
- 경기 중 프로그램 조작은 허용되지 않는다.
- 드론이 최종 착륙하기 전까지 데이터 전송 횟수에는 제한을 두지 않는다.
- 드론 착륙은 최초 이륙 지점으로 복귀하여 수행해야 한다.
#### 데이터 처리 흐름
 
1. 드론 영상/이미지 촬영
2. 전송 영상 선택
3. 저장 및 자동 PC 전송
4. AI 추론
5. 임무 코드 전송: JSON

## 12. 공통 임무코드 보고 템플릿
 
### 12.1 보고 규칙
 
- 모든 JSON 파일에는 `mission_code`를 반드시 포함한다.
- 전송 코드는 총 8개로 구성된다.
- 파일명은 지정된 형식을 그대로 사용한다.
- JSON 형식을 준수하며 UTF-8 인코딩으로 저장한다.
- 데이터가 없는 경우에도 필수 필드는 반드시 포함한다.
- 문자(String)와 숫자(Integer)를 구분하여 작성한다.
- `size`, `status`, `type`은 지정된 코드값만 입력한다.
- 준비 단계에서 `start.json`을 1회 전송한 후 임무를 시작한다.
- 임무 수행 중에는 모든 JSON 파일을 순서와 관계없이 전송할 수 있으며, 중복 전송도 가능하다.
- 임무 종료, 즉 드론 착륙 이전까지 수신된 데이터만 유효하다.
- 착륙 이후 전송된 데이터는 인정하지 않는다.
### 12.2 공통 필드
 
| 필드명 | 설명 | 예시 | 필수 여부 |
|---|---|---|---|
| `mission_code` | 팀을 구분하는 8자리 임무 코드 | `LKUSDC80` | 모든 JSON 파일에 포함 |
 
### 12.3 코드 종류 요약
 
#### 폭파구 크기 코드
 
| 코드 | 의미 |
|---|---|
| `big` | 대형 폭파구 |
| `medium` | 중형 폭파구 |
| `small` | 소형 폭파구 |
 
#### 시설물 상태 코드
 
| 코드 | 의미 |
|---|---|
| `normal` | 정상 |
| `destory` | 파손 |
| `fire` | 화재 |
 
주의: 문서에는 파손 코드가 `destory`로 표기되어 있다. 일반 영어 철자 `destroy`가 아니라, 대회 제공 명세의 코드명을 우선 확인해야 한다.

## 13. 종목 1: 폭파구 탐지 미션
 
- 최대 점수: 25점
### 13.1 폭파구 크기/위치 식별
 
- 배점: 15점
- 폭파구는 대형, 중형, 소형 총 3종류로 구성된다.
- 폭파구 위치 식별을 위해 경기장 내 구간 ID가 사전에 제공된다.
- 폭파구는 활주로와 유도로 구역에만 배치된다.
- 건물 구역에는 배치되지 않는다.
- 폭파구는 경기 당일 임의로 배치된다.
- 전체 30개 구간 중 5개 구간에 배치된다.
  - 활주로: 10개 구간
  - 유도로: 20개 구간
- 각 구간에는 최대 1개의 폭파구만 배치된다.
- 평가 방식: 활주로/유도로 구역 내 5개 배치 구간에 대해 각 구간별 3점
### 13.2 폭파구 개수 탐지
 
- 배점: 5점
- 폭파구의 크기와 종류를 구분하지 않고 개수를 산출한다.
- 활주로에 배치된 폭파구만 대상으로 한다.
- 유도로 구역의 폭파구는 개수 산정 대상에서 제외한다.
### 13.3 활주로 가용 길이 산출
 
- 배점: 5점
- 유도로 구역에 위치한 폭파구는 가용 길이 산정 대상에서 제외한다.
- 활주로 구역에서 폭파구가 존재하는 구간은 사용 불가능 구간으로 판단한다. 폭파구 크기는 가용 여부 판단에서 고려하지 않는다.
- 활주로 구역은 실제 환경 기준 총 3,000m이다.
- 활주로는 RW-01 ~ RW-10의 10개 구간으로 구분한다.
- 각 활주로 구간은 300m로 산정한다.
- 활주로의 가용 길이는 폭파구가 없는 구간 중 가장 긴 연속 운용 가능 구간을 기준으로 산정한다.
- 경기장 규모: 500cm × 400cm
- 경기장 축척: 1:600
- 실제 환경 규모: 약 3,000m × 2,400m
- 예시: 활주로 RW-01 구역의 실제 길이는 300m

## 14. 종목 2: 시설물 피해 확인 미션
 
- 최대 점수: 18점
### 14.1 시설물 상태 분류
 
- 배점: 시설물 6개, 각 3점
- 시설물 상태는 다음 3가지 유형으로 구분한다.
  - 정상
  - 파손
  - 화재
- 경기장 내 시설물은 총 6개로 구성된다.
- 참가자는 대회와 동일한 규격의 시설물이 배치된 연습 경기장을 통해 사전 학습할 수 있다.
- 시설물 위치는 변경되지 않으며, 상태만 변경된다.
- 각 시설물의 상태는 경기 당일 무작위로 결정된다.
- 시설물에 파손과 화재가 동시에 발생한 경우 화재 상태로 판정한다.
### 14.2 상태 정의
 
| 상태 | 정의 |
|---|---|
| 정상 | 외관 손상 및 화재가 없는 상태 |
| 파손 | 구조물의 일부가 파괴되거나 손상된 상태 |
| 화재 | 화염이 발생하며 일부 연기가 발생할 수 있는 상태 |
| 파손 + 화재 | 화재 상태로 판정 |

## 15. 종목 3: 위험 장애물 탐지 미션
 
- 최대 점수: 20점
### 15.1 불발탄 탐지
 
- 배점: 18점
- 탐지 대상은 미사일, 포탄, 자탄 총 3종류이다.
- 참가자는 대회와 동일한 규격의 불발탄 객체가 배치된 연습 경기장을 통해 사전 학습할 수 있다.
- 불발탄은 활주로 구역, 유도로 구역, 시설물 구역을 포함한 경기장 전역에 배치될 수 있다.
- 단, 활주로와 유도로 구역의 각 구간에 대해서만 위치/종류 결과를 제출하는 구조이다.
- 불발탄 위치 및 종류는 경기 당일 무작위로 결정된다.
- 구역별 배치 개수 제한은 없다.
- 단, 활주로와 유도로 6개 지정 구간에 배치되며, 각 구간에는 최대 1개의 불발탄만 배치된다.
- 평가 방식: 활주로/유도로 구역 내 6개 구간 배치, 각 구간별 3점
### 15.2 특정 구역 탐지
 
- 배점: 2점
- 활주로 구역에 배치된 불발탄의 총 개수를 탐지하여 제출해야 한다.
- 유도로 구역 및 시설물 구역에 배치된 불발탄은 집계 대상에서 제외한다.
- 불발탄의 종류, 즉 미사일/포탄/자탄과 관계없이 전체 개수를 산출한다.

## 16. 종목 4: LLM 기반 상황 보고
 
- 최대 점수: 7점
### 16.1 생성형 AI 활용
 
- 보고 항목:
  - 시간
  - 활주로 폭파구 개수
  - 가용 길이
  - 상태
  - 운용 여부
- 참가자는 생성형 AI, 즉 LLM을 활용하여 정찰 결과를 종합 분석하고 상황 보고서를 작성하여 제출해야 한다.
- 상황 보고서는 참가팀이 탐지한 임무 결과를 기반으로 자동 생성되어야 한다.
- 상황 보고서 작성에 사용되는 생성형 AI API 모델 종류는 제한하지 않는다.
- 상황 보고서는 지정된 글자 수 이내로 작성해야 한다.
  - 본문 설명에는 공백 포함 100자 이내로 제시되어 있다.
  - 평가표에는 50~100자 이내 조건이 표시되어 있다.
### 16.2 상황 보고서 필수 내용
 
상황 보고서에는 다음 내용을 포함해야 한다.
 
- 보고 시각: 년/월/일/시/분
- 활주로 구역 내 탐지된 폭파구 총 개수
- 가용 길이: 폭파구 탐지 미션의 가용 길이 산출 규칙을 따름
- 활주로 가용 길이에 따른 상태 및 운용 여부
- 보고 내용은 탐지한 결과를 기반으로 작성해야 하며, 확인되지 않은 내용은 포함하지 않는다.
### 16.3 활주로 가용 길이에 따른 상태/운용 여부
 
| 가용 길이 | 상태 | 운용 여부 |
|---:|---|---|
| 2,100m 이상 | 정상 | 사용 가능 |
| 1,500m 이상 ~ 2,100m 미만 | 제한 운용 | 제한적 사용 가능 |
| 900m 이상 ~ 1,500m 미만 | 비상 운용 | 사용 가능 여부 검토 |
| 900m 미만 | 운용 불가 | 사용 불가, 폐쇄 |
 
### 16.4 상황 보고서 작성 예시
 
예시 문장:
 
> 2026년 07월 16일 14시 27분 기준, 활주로 구역에서 폭파구 4개가 탐지되었으며 활주로 가용 길이는 1200미터로 산출된다. 현재 활주로는 비상 운용상태로 사용 가능 여부 검토가 필요하다.

## 19. 본선 과제의 핵심 해석
 
이 과제는 단순 이미지 탐지 모델 성능만 평가하는 문제가 아니라, 드론 운용, 영상 수집, AI 추론, 공간 좌표 변환, JSON 보고, LLM 상황보고, 시간 최적화가 결합된 시스템 통합형 임무이다.
 
특히 중요한 포인트는 다음과 같다.
 
1. 객체 탐지 정확도뿐 아니라 구간 ID 산출이 중요하다.
   - 폭파구와 불발탄은 단순 bbox 검출이 아니라 RW/TW/FA와 같은 경기장 구간으로 변환되어야 한다.
   - ArUco, homography, perspective transform, polygon ROI가 핵심이 될 가능성이 높다.
2. 활주로 가용 길이 산출은 룰 기반 후처리 문제이다.
   - 활주로 10개 구간 중 폭파구가 있는 구간을 unavailable로 보고, 가장 긴 연속 정상 구간 길이를 계산한다.
   - 각 활주로 구간은 실제 300m로 환산된다.
3. 시설물 미션은 위치가 고정되고 상태가 변하는 classification 성격이 강하다.
   - 시설물별 ROI를 잘 잡고 정상/파손/화재를 분류하는 전략이 유리하다.
   - 파손+화재는 화재로 판정해야 한다.
4. 불발탄 미션은 전체 경기장에 배치될 수 있으나 평가 제출은 활주로/유도로 구간 중심이다.
   - 위치와 종류가 모두 맞아야 한다.
   - 활주로 구역의 총 불발탄 개수도 별도 제출해야 한다.
5. LLM 보고는 탐지 결과 JSON을 자연어로 구조화하는 후처리 과제이다.
   - 실제 탐지되지 않은 내용을 넣으면 안 된다.
   - 보고 시간, 폭파구 개수, 가용 길이, 상태, 운용 여부가 필수이다.
6. 경기 중 프로그램 조작이 금지되므로 자동 파이프라인화가 필수이다.
   - 드론 촬영/저장/전송 이후 AI 추론과 JSON 송신이 자동으로 돌아가야 한다.
   - 임무 수행 중 노트북 조작은 경고 또는 중단/실격 사유가 될 수 있다.
7. 1차와 2차 평가 사이 약 10분의 수정 시간이 있다.
   - 1차 결과를 보고 빠르게 threshold, ROI, JSON 오류 등을 고치는 전략이 중요하다.
